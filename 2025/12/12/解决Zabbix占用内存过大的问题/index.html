解决Zabbix占用内存过大的问题

2025-12-12

通过优化PHP-FPM配置，解决Zabbix监控系统内存持续增长问题

问题背景

在使用Zabbix监控系统时，随着时间推移内存占用率会持续增长，甚至出现内存超过90%的告警情况（特别是虚拟机环境初始分配4G内存时）。

经定位发现这是由于PHP-FPM默认配置未优化导致的。

解决方案

通过优化PHP-FPM的进程管理配置，可显著降低内存占用。在Docker环境中，需要修改zabbix-web-nginx-mysql容器的PHP-FPM配置。

Docker环境优化步骤

1. 进入容器

docker exec -it –user root zabbix-web-nginx-mysql /bin/bash

2. 修改配置文件

# 备份原始配置文件
cp /etc/php-fpm.d/zabbix.conf /etc/php-fpm.d/zabbix.conf.bak

# 追加优化配置到文件末尾
cat >> /etc/php-fpm.d/zabbix.conf < ; 优化配置开始
pm = dynamic
pm.max_children = 16
pm.start_servers = 10
pm.min_spare_servers = 8
pm.max_spare_servers = 16
pm.max_requests = 2000
pm.process_idle_timeout = 10s
request_terminate_timeout = 120
; 优化配置结束
EOF

3. 退出并重启容器

# 退出容器
exit

# 重启容器使配置生效
docker restart zabbix-web-nginx-mysql

非Docker环境优化步骤（物理机/虚拟机）

1. 编辑配置文件

vi /etc/php-fpm.d/zabbix.conf

2. 添加优化配置

; 优化配置开始
pm = dynamic
pm.max_children = 16
pm.start_servers = 10
pm.min_spare_servers = 8
pm.max_spare_servers = 16
pm.max_requests = 2000
pm.process_idle_timeout = 10s
request_terminate_timeout = 120
; 优化配置结束

3. 重启PHP-FPM

systemctl restart php-fpm

配置参数说明

参数 推荐值 说明 优化效果
pm dynamic 进程管理方式：
- static: 固定进程数
- dynamic: 动态调整
- ondemand: 按需启动		 动态管理进程数，根据负载自动调整
pm.max_children 16 同一时刻允许的最大 PHP-FPM 子进程数 限制最大并发处理进程数
pm.start_servers 10 启动时的初始 PHP-FPM 进程数 避免启动时资源浪费
pm.min_spare_servers 8 空闲时最少保留的 PHP-FPM 进程数 保证基本响应能力
pm.max_spare_servers 16 空闲时最多保留的 PHP-FPM 进程数 限制空闲时资源占用
pm.max_requests 2000 每个子进程处理的最大请求数 定期重启进程释放内存
pm.process_idle_timeout 10s 空闲进程超时回收时间 及时回收空闲进程
request_terminate_timeout 120 单个请求最大执行时间 防止长时间占用进程

优化效果验证与监控建议

优化效果验证

  • 配置生效后，内存占用率应明显下降
  • 使用 docker stats 或 top 命令监控内存使用情况
  • 预期内存使用量稳定在1G左右（4G内存环境）

监控建议

  • 在Zabbix中配置内存使用率监控项：
    vm.memory.size[available]
    system.swap.size[,free]
  • 设置告警阈值（如内存使用率 > 80%）
  • 定期检查PHP-FPM进程状态：
    docker exec zabbix-web-nginx-mysql ps aux | grep php-fpm

注意事项

  • 根据实际服务器配置调整参数值：
    - 高配置服务器可适当增加 pm.max_children 值
    - 低配置服务器可适当降低 pm.max_children 值
  • 监控优化后的系统表现，必要时微调参数
  • 定期检查PHP-FPM日志：
    docker logs zabbix-web-nginx-mysql

备份与恢复

备份配置

docker cp zabbix-web-nginx-mysql:/etc/php-fpm.d/zabbix.conf ./zabbix.conf.bak

恢复配置

docker cp ./zabbix.conf.bak zabbix-web-nginx-mysql:/etc/php-fpm.d/zabbix.conf
docker restart zabbix-web-nginx-mysql

优化效果总结

通过以上优化配置，Zabbix系统的内存占用将得到有效控制，确保监控系统长期稳定运行。
