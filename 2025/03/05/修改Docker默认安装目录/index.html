五、注意事项

场景 处理方式
磁盘空间不足 确保 /data 分区有足够容量（建议 > 50GB）
SELinux 启用 添加 SELinux 标签：
sudo chcon -R system_u:object_r:container_file_t:s0 /data/docker
Firewalld 限制 开放必要端口：
sudo firewall-cmd –add-port=2375-2377/tcp –permanent
sudo firewall-cmd –reload
Kubernetes 集成 需同步修改 kubelet 的 –docker-root 参数
Docker Compose 项目 所有卷映射路径需基于新目录调整

六、配置解析

参数 作用 示例值
data-root 定义 Docker 数据存储根目录 /data/docker
exec-opts 存储驱动配置（可选） [“native.cgroupdriver=systemd”]
storage-driver 指定存储引擎（可选） overlay2

七、兼容性说明

系统环境 验证支持情况
CentOS/RHEL ✅ 全版本支持
Ubuntu/Debian ✅ 18.04+ 完美兼容
Kernel 版本 ≥ 3.10（推荐 ≥ 5.4）
Docker 版本 ≥ 19.03（推荐 24.0+）

通过以上步骤，Docker 的所有数据（包括镜像、容器、卷等）将永久存储到 /data/docker 目录。此方案适用于生产环境长期使用，如需还原配置，删除 daemon.json 中的 data-root 项并重启服务即可。
