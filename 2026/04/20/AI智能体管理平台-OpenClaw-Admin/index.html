AI智能体管理平台:OpenClaw-Admin

在飞书实现多 Agent，你不妨安装一个 Web 管理界面，通过这个界面来配置多 Agent，操作很简单，下面就讲讲怎么给 OpenClaw 装个 Web 管理界面。装完之后，打开浏览器就能管理一切，从而创建多Agent。

一、为什么要装 Web 界面

OpenClaw 自带一个 Dashboard，功能比较基础，能看状态、能聊天，但想管理模型配置、查看历史会话、切换智能体，就不太好用了。

OpenClaw-Admin 是个第三方开源项目，专门给 OpenClaw Gateway 做管理界面。功能全很多：

  • 📊 仪表盘：Token 用量趋势、会话统计、实时监控
  • 💬 在线对话：浏览器里直接跟 AI 聊，支持斜杠命令
  • 🤖 多智能体管理：创建、配置、切换不同的 AI 智能体
  • 📱 频道管理：QQ、飞书、钉钉、企微的可视化配置
  • 🔧 模型管理：API Key 配置、模型探测、默认模型设置
  • 📁 文件浏览器：工作区文件管理
  • 🖥️ 远程终端：SSE 远程终端，多节点支持

说白了，就是把命令行能干的事，全搬到浏览器里了。

二、项目介绍

这个项目叫 OpenClaw-Admin，开源在 GitHub：

项目地址：https://github.com/itq5/OpenClaw-Admin

核心功能：

功能模块 干啥用的
仪表盘 运行总览、Token 趋势、Top 模型/渠道分布
在线对话 实时聊天、斜杠命令、消息搜索
会话管理 会话列表、历史记录、导出
记忆管理 编辑 AGENTS、SOUL、IDENTITY 等核心文档
任务计划 定时任务、Cron 表达式、执行历史
模型管理 多模型配置、API Key 管理、模型探测
频道管理 QQ/飞书/钉钉/企微配置
技能管理 插件安装、更新、可见性控制
多智能体 创建智能体、配置权限、工作区管理
远程终端 SSE 终端、多节点、全屏模式
文件浏览器 工作区文件浏览、编辑、上传下载
系统监控 CPU/内存/磁盘、网络状态、运行时间

三、部署流程

如果你也想将该项目部署到本地使用的话，可以让OpenCode帮你安装和部署。

使用OpenCode

打开OpenCode，输入下面的命令：

学习这个项目：https://github.com/itq5/OpenClaw-Admin，并且帮我本地部署

接下来OpenCode就开始帮你本地部署这个项目，并且，出现报错时也可以帮你自动解决问题，无需担心不会代码。

访问管理界面

打开浏览器，访问：http://localhost:3001

登录信息：

  • 用户名：admin
  • 密码：admin

登录进去就能看到仪表盘了。

四、用途简介

简单介绍几个OpenClaw-Admin管理界面的用途。

仪表盘

仪表盘日常看你的大模型Token使用数，任务计划数等，相当于对你整个OpenClaw的使用做个总览。

记忆管理

在这里可以设置多角色，通过设置 AGENTS、SOUL、IDENTITY、USER、TOOLS、 HEARTBEAT、BOOTSTRAP、MEMORY文档内的内容，让每一个角色拥有不同的角色能力，形成一人公司（OPC）。

任务计划

可以新建或者取消任务。

模型管理

支持非官方推荐的其他模型，可以在这里手动配置。

频道管理

支持飞书、QQ、钉钉、企业微信等多个通道的配置。

技能管理

对已安装的技能进行管理。

智能体配置

分别对每一个智能体配置工具、模型和身份等。

虚拟公司

可视化一个虚拟公司的运行状态。
