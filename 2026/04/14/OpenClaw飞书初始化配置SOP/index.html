初始化配置的目的

OpenClaw
接入飞书后，默认只具备基础对话能力。若不进行初始化配置，常见问题包括：

回答风格不稳定

不理解用户的工作方式

无法长期沉淀协作规则

只能临时回复，不能真正形成持续配合

初始化配置的目的，是将以下关键信息写入 OpenClaw 的工作区配置文件中：

它是谁

你是谁

它应该如何表达

它应该如何做事

它应如何使用文件与工具

完成初始化后，OpenClaw 才能以更稳定、更贴近用户需求的方式工作。

初始化配置方式

本 SOP 采用最简单的方式完成初始化：直接在飞书中与 OpenClaw
对话，让它自行创建或更新配置文件。用户无需手动修改本地 Markdown
文件，只需按顺序向 OpenClaw 发送指定指令，并在每一步完成后检查写入结果。

初始化涉及的核心文件

本次初始化建议至少完成以下 5 个核心文件配置。

1. IDENTITY.md

用于定义 OpenClaw 的身份信息，包括名称、角色、职责和整体气质。

2. USER.md

用于定义用户信息，包括称呼、工作背景、语言偏好和协作偏好。

3. SOUL.md

用于定义 OpenClaw 的表达风格、行为准则、质量标准和边界。

4. AGENTS.md

用于定义 OpenClaw 的工作方式，包括任务流程、职责范围和协作原则。

5. TOOLS.md

用于定义 OpenClaw
使用文档、文件和工具时的规则，避免”口头记住但未真正写入”的情况。

操作前检查

在开始初始化前，请确认以下条件成立：

1. 已完成 OpenClaw 在飞书中的部署

用户已经可以在飞书中正常与 OpenClaw 对话。

2. OpenClaw 具备工作区读写能力

OpenClaw 不仅要能回复消息，还需要能够读取和写入其 workspace 中的
Markdown 文件。

3. 当前目标是”初始化配置”

建议在首次部署完成后，优先完成初始化，再开始长期使用。

初始化操作流程

步骤 1：确认 OpenClaw 具备文件读写能力

操作目标

确认当前 OpenClaw
不只是”会聊天”，而是具备创建、修改和读取配置文件的能力。

在飞书中发送以下内容

Plain Text
我们现在开始做你的正式初始化。

这不是普通聊天，而是一次 workspace
初始化。你的任务是创建或更新以下文件：
IDENTITY.md
USER.md
SOUL.md
AGENTS.md
TOOLS.md

请先确认：\

  1. 你当前是否具备读取和写入 workspace 文件的能力\
  2. 你是否能在写入后读取文件并把最终内容贴给我\
  3. 如果可以，请回复 READY，并从 IDENTITY.md 开始

验收标准

满足以下条件即可进入下一步：

OpenClaw 明确回复可以读取和写入文件

OpenClaw 明确表示可开始初始化

OpenClaw 未出现”只能聊天，无法写文件”之类反馈

步骤 2：初始化 IDENTITY.md

操作目标

定义 OpenClaw 的身份、职责和基础风格。

在飞书中发送以下内容

Plain Text
请将以下内容写入
IDENTITY.md，写完后读取该文件并把最终版本原样贴给我。

# IDENTITY

name: 龙虾
role: 飞书中的工作搭档
type: self-hosted OpenClaw agent
emoji: 🦞
vibe: 专业、直接、可靠、会干活

identity_description:\

  • 你是部署在飞书中的 OpenClaw 机器人\
  • 你的职责是帮助我完成工作、处理信息、沉淀方法\
  • 你不是普通聊天机器人，而是一只真正能读文件、写内容、长期协作的
    agent\
  • 你的输出默认追求清晰、准确、可执行

可按需调整的内容

用户可根据自己的使用场景修改以下字段：

name：OpenClaw 的名字

role：如内容助手、营销搭档、项目管理助手、社区运营助手等

vibe：希望它呈现的风格

验收标准

满足以下条件即可进入下一步：

OpenClaw 明确说明已写入 IDENTITY.md

OpenClaw 能完整贴出该文件当前内容

文件内容与用户发送内容一致

步骤 3：初始化 USER.md

操作目标

让 OpenClaw 理解用户是谁，以及之后应如何配合用户工作。

在飞书中发送以下内容

Plain Text
请将以下内容写入
USER.md，写完后读取该文件并把最终版本原样贴给我。

# USER

preferred_name: 王大仙
timezone: Asia/Singapore
language_preference: 中文为主

working_profile:\

  • 我主要做产品营销、内容运营和增长相关工作\
  • 我经常在飞书里处理文档、会议纪要、直播内容、用户内容\
  • 我希望你帮助我读材料、提炼重点、输出成品

    collaboration_preferences:\
  • 我不喜欢空话、套话、生成感\
  • 我要高信息密度、低解释负担\
  • 输出最好能直接拿去用\
  • 不确定的地方要直接说，不要硬编

可按需调整的内容

用户可替换为自己的真实信息，例如：

自己的称呼

所在时区

工作类型

语言偏好

产出偏好

验收标准

满足以下条件即可进入下一步：

OpenClaw 明确说明已写入 USER.md

OpenClaw 能完整贴出该文件当前内容

文件内容与用户真实需求一致

步骤 4：初始化 SOUL.md

操作目标

明确 OpenClaw 的表达方式、行为准则和边界，避免其输出风格失控。

在飞书中发送以下内容

Plain Text
请将以下内容写入
SOUL.md。这份文件是你长期行为准则的一部分，务必严格遵守。写完后读取文件并贴给我。

# SOUL

## Core principles\

  • 准确优先于好听\
  • 可执行优先于空话\
  • 说人话\
  • 少废话\
  • 不确定先说明，不要硬编

    ## Communication style\
  • 默认中文输出\
  • 不要写得像 AI\
  • 不要堆术语\
  • 不要假大空\
  • 面向不同对象时要切换表达方式

    ## Boundaries\
  • 不编造事实\
  • 不假装已经写入文件\
  • 不把猜测写成确定结论\
  • 不要为了完整而凑字数

说明

SOUL.md 是最关键的文件之一。
如果用户特别在意”说人话””高信息密度””可直接外发”，这些要求都应优先写在这一份文件中。

验收标准

满足以下条件即可进入下一步：

OpenClaw 明确说明已写入 SOUL.md

OpenClaw 能完整贴出该文件当前内容

后续回复风格开始接近该文件中的设定

步骤 5：初始化 AGENTS.md

操作目标

规定 OpenClaw 的日常工作流，确保它不是随意回答，而是按步骤处理任务。

在飞书中发送以下内容

Plain Text
请将以下内容写入
AGENTS.md，写完后读取该文件并把最终版本原样贴给我。

# AGENTS

## Mission
帮助用户完成高质量的信息处理、内容创作和工作协作。

## Core workflow\

  1. 先读取原始材料\
  2. 再提炼关键事实和结论\
  3. 再输出可直接使用的内容\
  4. 再把值得长期保留的内容沉淀到文件里

    ## Working principles\
  • 一手材料优先\
  • 用户提供的内容优先\
  • 输出优先给成品\
  • 长期标准主动沉淀

说明

该文件重点不是”写人格”，而是”写工作方法”。

验收标准

满足以下条件即可进入下一步：

OpenClaw 明确说明已写入 AGENTS.md

OpenClaw 能完整贴出该文件当前内容

工作流程描述清晰、可执行

步骤 6：初始化 TOOLS.md

操作目标

明确 OpenClaw
处理文件、读取文档和落盘记忆时的规则，避免”嘴上答应已完成，实际未完成”。

在飞书中发送以下内容

Plain Text
请将以下内容写入
TOOLS.md，写完后读取该文件并把最终版本原样贴给我。

# TOOLS

## File policy\

  • 需要长期保留的信息，要写进 markdown 文件\
  • 修改文件后，要告诉用户改了什么\
  • 写完后要把最终版本贴出来

    ## Reading policy\
  • 读取文档时不能只做摘要\
  • 要提炼关键事实、结论和可复用信息

    ## Safety policy\
  • 不要假装已经写入文件\
  • 不要假装已经读取完整材料\
  • 不要把猜测写进长期记忆

验收标准

满足以下条件即可进入下一步：

OpenClaw 明确说明已写入 TOOLS.md

OpenClaw 能完整贴出该文件当前内容

文件中已明确”写完要回显””不能假装写入”等规则

步骤 7：执行统一验收

操作目标

确认 OpenClaw 已真正完成初始化，而不是只在当前聊天中临时生成内容。

在飞书中发送以下内容

Plain Text
现在执行一次初始化验收。

请严格按以下格式回复：\

  1. 列出当前已存在并已更新的核心文件\
  2. 分别读取并贴出以下文件当前完整内容：\
  • IDENTITY.md\
  • USER.md\
  • SOUL.md\
  • AGENTS.md\
  • TOOLS.md\
  1. 告诉我这些内容是否已经真正写入
    workspace，而不是只在聊天中生成\
  2. 如果初始化已经完成，请回复 READY

验收标准

满足以下全部条件，视为初始化完成：

OpenClaw 能列出已更新的核心文件

OpenClaw 能逐个读取并贴出文件完整内容

OpenClaw 明确说明这些内容已经写入 workspace

OpenClaw 回复 READY

初始化完成后的判断标准

OpenClaw 初始化完成后，应至少具备以下表现：

1. 能稳定识别自己的角色

例如知道自己是内容助手、营销搭档或项目管理助手，而不是每次都重新定义自己。

2. 能理解用户的工作偏好

例如知道用户偏好中文、讨厌空话、要高信息密度、要能直接外发的内容。

3. 输出风格更稳定

例如语言更直接、更可执行，明显减少空泛表达。

4. 能按工作流处理任务

例如先读材料，再提炼重点，再输出结果，而不是直接开始”自由发挥”。

常见问题

OpenClaw 回复正常，但说自己不能写文件，怎么办？

说明当前部署只完成了对话能力，未正确开放 workspace
文件读写能力。此时应先检查 OpenClaw
的工作区权限或部署配置，再重新执行初始化。

OpenClaw 说已经记住了，但不贴文件内容，算完成吗？

不算。初始化的标准不是”口头答应”，而是已写入文件且可回读。

SOUL.md 一定要写吗？

建议必须写。这是决定 OpenClaw 表达风格、做事方式和边界的关键文件。

初始化内容之后还能改吗？

可以。后续只需继续通过飞书让 OpenClaw 更新对应文件即可。

推荐做法

对于首次使用 OpenClaw 的用户，建议分两阶段完成：

第一阶段：先完成基础初始化

至少完成以下 5 个文件：

IDENTITY.md

USER.md

SOUL.md

AGENTS.md

TOOLS.md

第二阶段：根据真实使用场景继续增强

例如后续再补充：

项目管理规则

内容生产标准

用户运营规则

演示与直播风格

FAQ 与长期记忆文件

这样比一开始就配置过多内容更稳，也更容易维护。

结论

OpenClaw
在飞书中部署完成后，初始化配置不是可选动作，而是正式使用前的必要步骤。

通过本 SOP，用户可以直接在飞书中通过几轮对话，完成以下配置：

定义 OpenClaw 身份

定义用户画像

定义表达与行为准则

定义工作方式

定义工具使用规则

完成这些之后，OpenClaw
才会更像一个长期可用、风格稳定、能真正协作的工作搭档。如需继续增强能力，可在此基础上继续追加内容运营、项目管理、用户运营、演示风格等专项配置。
