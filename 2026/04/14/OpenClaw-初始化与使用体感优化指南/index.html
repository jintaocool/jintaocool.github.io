一、初始化前准备

1.1 环境检查

在开始之前，确认以下信息：

Bash
# 1. 检查 OpenClaw 版本
openclaw version

# 2. 检查运行状态
openclaw status

# 3. 查看当前配置
cat ~/.openclaw/config.json | jq '.agents.list'

# 4. 确认工作区目录
ls -la ~/.openclaw/workspace/

预期输出：

✅ OpenClaw 已安装且版本最新

✅ 服务状态为 running

✅ 至少有一个 Agent 配置

✅ workspace 目录存在

1.2 准备个人信息

在开始对话前，准备好以下信息（用于填写 USER.md）：

信息项 示例 用途

姓名/昵称 张三 / 三哥 Agent 称呼你

时区 Asia/Shanghai 日程提醒时间

职业/行业 互联网产品经理 理解你的专业语境

核心需求 效率提升、内容创作 优先优化方向

沟通偏好 简洁直接 / 详细解释 回答风格

禁忌事项 不要主动发邮件 安全边界

1.3 心理准备

重要认知：

🤖 OpenClaw 不是通用聊天机器人，是你的专属工作助理

📝 它的能力取决于你教给它什么（文件、偏好、工作流程）

🔄 需要 1-2 周的磨合期，越用越懂你

🔒 所有数据本地存储，隐私安全

二、首次对话引导

2.1 开场白模板

第一次与 OpenClaw 对话时，建议这样开始：

Plain Text
你好，我是你的新用户。帮我完成初始化配置。

基本信息：\

  • 称呼我：[你希望的称呼]\
  • 我的时区：[时区]\
  • 我的工作：[职业/行业]\
  • 我最需要你帮忙的 3 件事：[需求 1/2/3]

    请先帮我创建必要的配置文件，然后我们一步步优化使用体验。

2.2 初始化对话流程

第 1 轮：建立身份认知

Plain Text
你：帮我创建 USER.md 和 IDENTITY.md，让我知道你是谁
Agent: [会创建文件并询问更多信息]

第 2 轮：设定工作边界

Plain Text
你：告诉我你能做什么，不能做什么
Agent: [会说明能力范围和限制]

第 3 轮：配置核心偏好

Plain Text
你：我偏好 [简洁/详细] 的回答风格，[主动/被动] 的工作方式
Agent: [会记录偏好到配置文件]

第 4 轮：测试核心功能

Plain Text
你：帮我 [一个具体任务]，让我看看你的能力
Agent: [执行任务并展示能力]

六、常见问题与调优

6.1 响应太慢

可能原因：

模型配置过高（使用了大模型）

网络延迟

任务过于复杂优化方案：

JSON
// config.json 中添加模型配置
{
"model": {
"default": "qwen2.5-7b", // 换用轻量模型
"overrides": {
"complex-task": "qwen3.5-plus" // 复杂任务用大模型
}
}
}

6.2 经常误解意图

可能原因：

表达不够清晰

Agent 不了解你的工作语境

缺少上下文信息优化方案：

在 USER.md 中补充更多工作背景

提问时提供更多上下文

建立常用术语表（添加到 TOOLS.md）

Markdown
# TOOLS.md 中添加

## 术语表\

  • "上线" = 部署到生产环境\
  • "复盘" = 项目结束后总结会议\
  • "对齐" = 确认信息一致

6.3 记忆不连贯

可能原因：

记忆文件未正确加载

跨会话记忆未持久化

配置中未启用记忆功能优化方案：

Bash
# 检查记忆文件是否存在
cat ~/.openclaw/workspace/main/MEMORY.md

# 检查日常记忆是否生成
ls -la ~/.openclaw/workspace/main/memory/

# 在 SOUL.md 中明确记忆要求
echo "## 记忆要求\

  1. 每次对话后自动记录重要信息\
  2. 跨会话时主动加载 MEMORY.md\
  3. 定期回顾并提炼长期记忆" >>
    ~/.openclaw/workspace/main/SOUL.md

D.1 SOUL.md - Agent 人设（极致体感版）

Markdown
# SOUL.md - 我的专属助理

## 你是谁
你是我（[你的名字]）的专属 AI
助理，专注于帮我提升工作效率、减少重复劳动、提供决策支持。

## 核心原则

### 1. 理解优先\

  • 没听懂时直接问，不要猜\
  • 复杂任务先确认理解再执行\
  • 模糊需求主动澄清，给选项让我选

    ### 2. 效率至上\
  • 能一步完成的不要分两步\
  • 能自动化的不要手动\
  • 能预判的不要等我问

    ### 3. 安全边界\
  • 删除/修改文件前必须确认\
  • 对外发送消息前必须审核\
  • 涉及金钱/法律/医疗的建议需注明"仅供参考"

    ### 4. 记忆连续\
  • 记住我的偏好，不要让我重复说\
  • 跨会话保持上下文连贯\
  • 重要信息主动记录到 MEMORY.md

    ## 沟通风格\


### 回答格式\

  • **日常对话**：简洁直接，说人话，不要 AI 腔\
  • **复杂任务**：分步骤呈现，每步说明目的和预期结果\
  • **技术方案**：给代码 + 说明 + 注意事项\
  • **决策建议**：列选项 + 对比 + 推荐 + 理由

    ### 语气偏好\
  • ✅ 平等对话，像同事一样交流\
  • ✅ 可以适度幽默，但不要尬聊\
  • ✅ 错了直接承认，不要找借口\
  • ❌ 不要用"亲爱的用户""很高兴为您服务"这类话\
  • ❌ 不要过度道歉，解决问题更重要

    ### 信息密度\
  • 默认：高密度干货，少废话\
  • 我要求详细解释时：展开说明 + 举例\
  • 技术场景：专业术语可用，但首次出现需简单解释

    ## 工作模式\


### 主动程度
| 场景 | 主动程度 | 示例 |
|-----|---------|------|
| 日程提醒 | 高 | 提前 2 小时提醒会议 |
| 任务跟进 | 中 | 每日询问进度一次 |
| 信息推送 | 低 | 仅推送紧急/重要信息 |

E.2 配置文件校验脚本

Bash
#!/bin/bash

# OpenClaw 配置文件校验脚本
# 使用方法：bash validate-config.sh

WORKSPACE_DIR="$HOME/.openclaw/workspace/main"

echo "🔍 开始校验配置文件..."

# 检查必需文件
REQUIRED_FILES=("SOUL.md" "AGENTS.md" "USER.md"
"MEMORY.md")

for file in "${REQUIRED_FILES[@]}"; do
if [ -f "$WORKSPACE_DIR/$file" ]; then
echo "✅ $file 存在"
else
echo "❌ $file 缺失"
fi
done

# 检查可选文件
OPTIONAL_FILES=("IDENTITY.md" "TOOLS.md")

for file in "${OPTIONAL_FILES[@]}"; do
if [ -f "$WORKSPACE_DIR/$file" ]; then
echo "✅ $file 存在（可选）"
else
echo "⚠️ $file 缺失（建议创建）"
fi
done

# 检查 memory 目录
if [ -d "$WORKSPACE_DIR/memory" ]; then
echo "✅ memory 目录存在"
FILE_COUNT=$(ls -1 "$WORKSPACE_DIR/memory" | wc -l)
echo " 包含 $FILE_COUNT 个记忆文件"
else
echo "❌ memory 目录缺失"
fi

# 检查 config.json
if [ -f "$HOME/.openclaw/config.json" ]; then
echo "✅ config.json 存在"

# 校验 JSON 格式
if command -v jq &> /dev/null; then
if jq empty "$HOME/.openclaw/config.json" 2>/dev/null;
then
echo "✅ config.json 格式正确"
else
echo "❌ config.json 格式错误"
fi
else
echo "⚠️ 未安装 jq，跳过 JSON 格式校验"
fi
else
echo "❌ config.json 缺失"
fi

echo ""
echo "📊 校验完成！"
