Prometheus开启basic_auth认证

2024-08-08
Prometheus
字数统计: 795   |   阅读时长≈ 4 分钟

考虑将公司的联邦集群（pull）换成remote_write（push）这种形式， 所以需要将Prometheus开放到公网，看了看认证相关的配置

也可以使用Nginx来反向代理，可以参考 Nginx开启基本http认证， 不过Prometheus原生带了basic auth 和ssl认证, 官网的说明https://prometheus.io/docs/guides/basic-auth/

开启web配置文件
plain<br>1<br> plain<br>./prometheus --web.config.file="web.yml" --web.listen-address="0.0.0.0:9001"<br>

生成密码

密码需要bcrypt加密，这里使用htpasswd工具生成

Ubuntu和Debian安装CentOS和Fedora安装
plain<br>1<br> plain<br>apt install apache2-utils<br>
plain<br>1<br> plain<br>htpasswd -nB 'admin'<br>

web配置文件

vim web.yml
plain<br>1<br>2<br> plain<br>basic_auth_users:<br> admin: $2y$05$UKSS18ztdsUNoEuXYScr2OE1TCMe1hWnmD6JuwUi/uPTJayHIakae<br>

验证一下配置
plain<br>1<br> plain<br>promtool check web-config web.yml<br>

remote write 访问

后面访问控制台页面或者调用接口都需要指定账号密码了
plain<br>1<br> plain<br>curl -u admin:xxxxx http://localhost:9090<br>

其他Prometheus如果需要往这台写入， 需要在其他的Prometheus remote_write配置认证
plain<br>1<br>2<br>3<br>4<br>5<br>6<br>7<br>8<br>9<br>10<br>11<br>12<br>13<br>14<br>15<br>16<br> plain<br>remote_write:<br>- url: "http://127.0.0.1:9090/api/v1/write"<br> basic_auth:<br> username: admin<br> password: xxxxx<br> remote_timeout: 30s<br> tls_config:<br> insecure_skip_verify: true<br> queue_config:<br> capacity: 500<br> max_shards: 1000<br> min_shards: 1<br> max_samples_per_send: 100<br> batch_send_deadline: 5s<br> min_backoff: 30ms<br> max_backoff: 100ms<br>

联邦配置

如果此Prometheus需要通过联邦机制接入到其他Prometheus， 需要在对方的配置里面增加：
plain<br>1<br>2<br>3<br>4<br>5<br>6<br>7<br>8<br>9<br>10<br>11<br>12<br>13<br>14<br>15<br> plain<br>- job_name: 'federate-http'<br> scrape_interval: 30s<br> scrape_timeout: 30s<br> honor_labels: true<br> metrics_path: '/federate'<br> basic_auth:<br> username: 'admin'<br> password: 'xxxxx'<br> params:<br> 'match[]':<br> - '{job=~".+"}'<br> file_sd_configs:<br> - files:<br> - /data/prometheus/conf.d/federate_http.yml<br> refresh_interval: 30s<br>

启用了Basic Auth怎么健康检查

在Kubernetes环境下，配置如下：
plain<br>1<br>2<br> plain<br># 生成base64的账号密码编码<br>echo -n "user:pass" |base64 -w0<br>

在 Liveness 和 Readness 配置
plain<br>1<br>2<br>3<br>4<br> plain<br>httpGet:<br> httpHeaders:<br> - name: Authorization<br> value: Basic dXNlcjpwYXNz<br>
